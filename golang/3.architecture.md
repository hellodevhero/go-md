# Week 1 — Clean Architecture

## 1.1 Why Architecture Matters

In Phase 2 you built a working API. The handlers called the store directly. The store
used `*sql.DB` directly. It worked, but there are problems:

**The "before" — everything mixed together:**

```go
// BAD: handler contains SQL, business logic, and HTTP concerns all in one place
func createOrder(w http.ResponseWriter, r *http.Request) {
    var req OrderRequest
    json.NewDecoder(r.Body).Decode(&req)

    // Business logic mixed with SQL
    var stock int
    db.QueryRow("SELECT stock FROM books WHERE id = $1", req.BookID).Scan(&stock)
    if stock < req.Quantity {
        http.Error(w, "insufficient stock", 400)
        return
    }
    db.Exec("UPDATE books SET stock = stock - $1 WHERE id = $2", req.Quantity, req.BookID)
    db.Exec("INSERT INTO orders (book_id, quantity) VALUES ($1, $2)", req.BookID, req.Quantity)

    w.WriteHeader(201)
}
// Problems: untestable without DB, business rules buried in SQL, hard to reuse logic
```

Here are the specific problems:

- You cannot test handlers without a real database
- Swapping PostgreSQL for SQLite (for tests) requires rewriting everything
- Adding a new feature touches too many files
- Business logic is mixed into SQL queries

Clean architecture solves this by enforcing strict dependency rules. The core idea:
**inner layers know nothing about outer layers.**

```
┌─────────────────────────────────────────┐
│  Delivery (HTTP handlers, CLI)          │  ← knows about Use Cases
├─────────────────────────────────────────┤
│  Use Cases (business logic)             │  ← knows about Entities & Repos
├─────────────────────────────────────────┤
│  Repository interfaces                  │  ← knows about Entities only
├─────────────────────────────────────────┤
│  Entities (domain models)               │  ← knows nothing else
└─────────────────────────────────────────┘
```

Dependencies point inward only. The database is a detail — the business logic
does not care if it is PostgreSQL, SQLite, or an in-memory map.

## 1.2 Layers Explained

### Layer 1 — Domain / Entities

Pure Go structs and business rules. No imports from your other packages, no
database tags, no JSON tags. This is the heart of your application.

```go
// internal/domain/book.go
package domain

import (
    "errors"
    "time"
)

// Errors are defined in the domain — not in the database layer
var (
    ErrBookNotFound    = errors.New("book not found")
    ErrOutOfStock      = errors.New("book is out of stock")
    ErrInvalidPrice    = errors.New("price must be greater than zero")
    ErrInvalidTitle    = errors.New("title cannot be empty")
)

// Book is a pure domain entity — no db/json tags
type Book struct {
    ID          int
    Title       string
    Author      string
    Description string
    Price       float64
    Stock       int
    Year        int
    CreatedAt   time.Time
}

// Validate encapsulates business rules about what makes a valid book
func (b *Book) Validate() error {
    if b.Title == "" {
        return ErrInvalidTitle
    }
    if b.Price <= 0 {
        return ErrInvalidPrice
    }
    return nil
}

// IsAvailable is a business rule — lives on the entity, not in a handler
func (b *Book) IsAvailable() bool {
    return b.Stock > 0
}

// Reserve reduces stock — another business rule on the entity
func (b *Book) Reserve(qty int) error {
    if b.Stock < qty {
        return ErrOutOfStock
    }
    b.Stock -= qty
    return nil
}
```

```go
// internal/domain/order.go
package domain

import "time"

var (
    ErrOrderNotFound  = errors.New("order not found")
    ErrEmptyOrder     = errors.New("order must have at least one item")
)

type OrderStatus string

const (
    OrderStatusPending   OrderStatus = "pending"
    OrderStatusConfirmed OrderStatus = "confirmed"
    OrderStatusShipped   OrderStatus = "shipped"
    OrderStatusCancelled OrderStatus = "cancelled"
)

type OrderItem struct {
    BookID   int
    Quantity int
    Price    float64 // price at time of order — never recalculate
}

type Order struct {
    ID        int
    UserID    int
    Items     []OrderItem
    Total     float64
    Status    OrderStatus
    CreatedAt time.Time
}

func NewOrder(userID int, items []OrderItem) (*Order, error) {
    if len(items) == 0 {
        return nil, ErrEmptyOrder
    }

    var total float64
    for _, item := range items {
        total += item.Price * float64(item.Quantity)
    }

    return &Order{
        UserID: userID,
        Items:  items,
        Total:  total,
        Status: OrderStatusPending,
    }, nil
}
```

### Layer 2 — Repository Interfaces

Interfaces that describe what the business logic needs from storage.
No SQL, no `*sql.DB`, no PostgreSQL-specific anything.

```go
// internal/domain/repository.go
package domain

import "context"

// BookRepository defines what storage operations the use cases need.
// The use cases depend on this interface — not on any concrete implementation.
type BookRepository interface {
    Create(ctx context.Context, book *Book) (*Book, error)
    GetByID(ctx context.Context, id int) (*Book, error)
    List(ctx context.Context, filter BookFilter) ([]Book, int, error)
    Update(ctx context.Context, book *Book) (*Book, error)
    Delete(ctx context.Context, id int) error
    UpdateStock(ctx context.Context, id, delta int) error
}

type UserRepository interface {
    Create(ctx context.Context, user *User) (*User, error)
    GetByID(ctx context.Context, id int) (*User, error)
    GetByEmail(ctx context.Context, email string) (*User, error)
}

type OrderRepository interface {
    Create(ctx context.Context, order *Order) (*Order, error)
    GetByID(ctx context.Context, id int) (*Order, error)
    ListByUserID(ctx context.Context, userID int) ([]Order, error)
    UpdateStatus(ctx context.Context, id int, status OrderStatus) error
}

type BookFilter struct {
    Search   string
    Author   string
    MinPrice float64
    MaxPrice float64
    Page     int
    Limit    int
```

### Layer 3 — Use Cases (Business Logic)

Use cases orchestrate domain objects and call repository interfaces.
They contain the *what* of your application — not the *how*.

```go
// internal/usecase/book.go
package usecase

import (
    "context"
    "fmt"

    "github.com/yourname/bookstore/internal/domain"
)

type BookUseCase struct {
    repo  domain.BookRepository
    cache BookCache // interface — could be Redis or a no-op
}

// BookCache is a thin interface — only what we need
type BookCache interface {
    Get(ctx context.Context, key string) (*domain.Book, error)
    Set(ctx context.Context, key string, book *domain.Book) error
    Delete(ctx context.Context, key string) error
}

func NewBookUseCase(repo domain.BookRepository, cache BookCache) *BookUseCase {
    return &BookUseCase{repo: repo, cache: cache}
}

func (uc *BookUseCase) Create(ctx context.Context, book *domain.Book) (*domain.Book, error) {
    // Business rule: validate before saving
    if err := book.Validate(); err != nil {
        return nil, fmt.Errorf("create book: %w", err)
    }

    created, err := uc.repo.Create(ctx, book)
    if err != nil {
        return nil, fmt.Errorf("create book: %w", err)
    }
    return created, nil
}

func (uc *BookUseCase) GetByID(ctx context.Context, id int) (*domain.Book, error) {
    cacheKey := fmt.Sprintf("book:%d", id)

    // Try cache first
    if book, err := uc.cache.Get(ctx, cacheKey); err == nil {
        return book, nil
    }

    book, err := uc.repo.GetByID(ctx, id)
    if err != nil {
        return nil, fmt.Errorf("get book: %w", err)
    }

    // Populate cache for next time
    _ = uc.cache.Set(ctx, cacheKey, book)

    return book, nil
}

func (uc *BookUseCase) Update(ctx context.Context, book *domain.Book) (*domain.Book, error) {
    if err := book.Validate(); err != nil {
        return nil, fmt.Errorf("update book: %w", err)
    }

    updated, err := uc.repo.Update(ctx, book)
    if err != nil {
        return nil, fmt.Errorf("update book: %w", err)
    }

    // Invalidate cache
    _ = uc.cache.Delete(ctx, fmt.Sprintf("book:%d", book.ID))

    return updated, nil
}

func (uc *BookUseCase) List(ctx context.Context, filter domain.BookFilter) ([]domain.Book, int, error) {
    if filter.Limit == 0 {
        filter.Limit = 20
    }
    if filter.Page < 1 {
        filter.Page = 1
    }
    return uc.repo.List(ctx, filter)
}

func (uc *BookUseCase) Delete(ctx context.Context, id int) error {
    if err := uc.repo.Delete(ctx, id); err != nil {
        return fmt.Errorf("delete book: %w", err)
    }
    _ = uc.cache.Delete(ctx, fmt.Sprintf("book:%d", id))
    return nil
}
```

```go
// internal/usecase/order.go
package usecase

import (
    "context"
    "fmt"

    "github.com/yourname/bookstore/internal/domain"
)

type OrderUseCase struct {
    orderRepo domain.OrderRepository
    bookRepo  domain.BookRepository
}

func NewOrderUseCase(orderRepo domain.OrderRepository, bookRepo domain.BookRepository) *OrderUseCase {
    return &OrderUseCase{orderRepo: orderRepo, bookRepo: bookRepo}
}

type CreateOrderInput struct {
    UserID int
    Items  []struct {
        BookID   int
        Quantity int
    }
}

func (uc *OrderUseCase) Create(ctx context.Context, input CreateOrderInput) (*domain.Order, error) {
    // Resolve books and check stock — all business logic here, not in handler
    var items []domain.OrderItem
    for _, item := range input.Items {
        book, err := uc.bookRepo.GetByID(ctx, item.BookID)
        if err != nil {
            return nil, fmt.Errorf("book %d: %w", item.BookID, err)
        }

        // Domain method enforces the business rule
        if err := book.Reserve(item.Quantity); err != nil {
            return nil, fmt.Errorf("book %q: %w", book.Title, err)
        }

        items = append(items, domain.OrderItem{
            BookID:   book.ID,
            Quantity: item.Quantity,
            Price:    book.Price, // capture price at order time
        })
    }

    // Domain constructor validates the order
    order, err := domain.NewOrder(input.UserID, items)
    if err != nil {
        return nil, err
    }

    // Persist — in a real app this would be a transaction
    created, err := uc.orderRepo.Create(ctx, order)
    if err != nil {
        return nil, fmt.Errorf("create order: %w", err)
    }

    // Update stock for each book
    for _, item := range input.Items {
        _ = uc.bookRepo.UpdateStock(ctx, item.BookID, -item.Quantity)
    }

    return created, nil
}
```

### Layer 4 — Repository Implementations (Infrastructure)

Concrete implementations of the repository interfaces.
This is where SQL lives. Only this layer knows about PostgreSQL.

```go
// internal/repository/postgres/book.go
package postgres

import (
    "context"
    "database/sql"
    "fmt"
    "strings"

    "github.com/yourname/bookstore/internal/domain"
)

type BookRepository struct {
    db *sql.DB
}

func NewBookRepository(db *sql.DB) *BookRepository {
    return &BookRepository{db: db}
}

// Implement domain.BookRepository interface
func (r *BookRepository) GetByID(ctx context.Context, id int) (*domain.Book, error) {
    var b domain.Book
    err := r.db.QueryRowContext(ctx,
        `SELECT id, title, author, description, price, stock, year, created_at
         FROM books WHERE id = $1`, id,
    ).Scan(&b.ID, &b.Title, &b.Author, &b.Description,
        &b.Price, &b.Stock, &b.Year, &b.CreatedAt)

    if err == sql.ErrNoRows {
        return nil, domain.ErrBookNotFound
    }
    if err != nil {
        return nil, fmt.Errorf("postgres getBook: %w", err)
    }
    return &b, nil
}

func (r *BookRepository) Create(ctx context.Context, b *domain.Book) (*domain.Book, error) {
    err := r.db.QueryRowContext(ctx,
        `INSERT INTO books (title, author, description, price, stock, year)
         VALUES ($1, $2, $3, $4, $5, $6)
         RETURNING id, created_at`,
        b.Title, b.Author, b.Description, b.Price, b.Stock, b.Year,
    ).Scan(&b.ID, &b.CreatedAt)

    if err != nil {
        return nil, fmt.Errorf("postgres createBook: %w", err)
    }
    return b, nil
}

// UpdateStock uses an atomic SQL update — no read-modify-write race
func (r *BookRepository) UpdateStock(ctx context.Context, id, delta int) error {
    _, err := r.db.ExecContext(ctx,
        `UPDATE books SET stock = stock + $1 WHERE id = $2`,
        delta, id,
    )
    return err
}
```

### Layer 5 — Delivery (HTTP Handlers)

Handlers only translate HTTP ↔ use case. No business logic here.

```go
// internal/delivery/http/book.go
package http

import (
    "encoding/json"
    "errors"
    "net/http"
    "strconv"

    "github.com/yourname/bookstore/internal/domain"
    "github.com/yourname/bookstore/internal/usecase"
)

type BookHandler struct {
    uc *usecase.BookUseCase
}

func NewBookHandler(uc *usecase.BookUseCase) *BookHandler {
    return &BookHandler{uc: uc}
}

func (h *BookHandler) Create(w http.ResponseWriter, r *http.Request) {
    var input domain.Book
    if err := json.NewDecoder(r.Body).Decode(&input); err != nil {
        writeError(w, http.StatusBadRequest, "invalid request body")
        return
    }

    book, err := h.uc.Create(r.Context(), &input)
    if err != nil {
        // Map domain errors to HTTP status codes — this is the handler's job
        switch {
        case errors.Is(err, domain.ErrInvalidTitle),
             errors.Is(err, domain.ErrInvalidPrice):
            writeError(w, http.StatusBadRequest, err.Error())
        default:
            writeError(w, http.StatusInternalServerError, "internal error")
        }
        return
    }

    writeJSON(w, http.StatusCreated, book)
}

func (h *BookHandler) GetByID(w http.ResponseWriter, r *http.Request) {
    id, err := strconv.Atoi(r.PathValue("id"))
    if err != nil {
        writeError(w, http.StatusBadRequest, "invalid book ID")
        return
    }

    book, err := h.uc.GetByID(r.Context(), id)
    if err != nil {
        if errors.Is(err, domain.ErrBookNotFound) {
            writeError(w, http.StatusNotFound, "book not found")
            return
        }
        writeError(w, http.StatusInternalServerError, "internal error")
        return
    }

    writeJSON(w, http.StatusOK, book)
}
```

## 1.4 Dependency Injection — Wiring it Together

**Dependency Injection (DI)** means passing dependencies into a struct from the outside instead of creating them internally. Instead of a use case calling `postgres.NewBookRepository(db)` itself, it receives a `domain.BookRepository` interface through its constructor. The caller decides what concrete implementation to provide.

**The composition root** is the single place in your application where all concrete implementations are created and wired together — typically `main.go`. This is the ONLY file that imports `postgres`, `rediscache`, and other infrastructure packages. Every other package depends only on interfaces.

**Why this works:** Go's type system enforces interface satisfaction at compile time. If you wire a `postgres.BookRepository` where a `domain.BookRepository` is expected, the compiler checks that every method exists. A wiring mistake is a compile error, not a runtime error.

`main.go` is the composition root — the only place that knows about all layers:

```go
// cmd/api/main.go
func main() {
    cfg := config.MustLoad()

    // Infrastructure
    db       := mustConnectDB(cfg.DatabaseURL)
    redisClient := mustConnectRedis(cfg.RedisURL)

    // Repositories (implement domain interfaces)
    bookRepo  := postgres.NewBookRepository(db)
    userRepo  := postgres.NewUserRepository(db)
    orderRepo := postgres.NewOrderRepository(db)
    bookCache := rediscache.NewBookCache(redisClient)

    // Use cases (depend on interfaces — not on postgres/redis)
    bookUC  := usecase.NewBookUseCase(bookRepo, bookCache)
    userUC  := usecase.NewUserUseCase(userRepo, cfg.JWTSecret)
    orderUC := usecase.NewOrderUseCase(orderRepo, bookRepo)

    // Handlers (depend on use cases)
    bookHandler  := delivery.NewBookHandler(bookUC)
    userHandler  := delivery.NewUserHandler(userUC)
    orderHandler := delivery.NewOrderHandler(orderUC)

    // Router
    router := delivery.NewRouter(bookHandler, userHandler, orderHandler, cfg)
    runServer(router, cfg)
}
```

**The payoff:** To test `BookUseCase`, you pass a fake `BookRepository` that returns
hardcoded data. No database needed. Tests run instantly.

# Week 2 — Gin Framework

## 2.1 Why Gin

Go's standard `net/http` is powerful but verbose for large APIs:
routing by method+path requires boilerplate, path parameters need manual parsing,
and validation must be written from scratch.

**Gin** solves these without hiding what Go is doing. It is thin, fast, and the
most widely used Go HTTP framework.

**How Gin compares to alternatives:**
- **`chi`** — closest to `net/http`, uses standard `http.Handler`. No framework lock-in. Great if you want minimal dependencies.
- **`echo`** — similar feature set to Gin (routing, middleware, binding). Slightly different API style. Smaller community.
- **`fiber`** — uses `fasthttp` instead of `net/http`. Very fast for benchmarks, but breaks compatibility with the standard library ecosystem (`http.Handler`, middleware, etc.).
- **Standard `net/http`** (Go 1.22+) — the improved `ServeMux` now supports method routing and path parameters, narrowing the gap significantly. For simple APIs, you may not need a framework at all.

We use Gin here because it is the pragmatic middle ground: familiar to most Go developers, well-documented, and provides just enough convenience (binding, validation, middleware chaining) without hiding the underlying HTTP model.

```bash
go get github.com/gin-gonic/gin
```

## 2.2 Basic Gin

```go
package main

import (
    "net/http"
    "github.com/gin-gonic/gin"
)

func main() {
    r := gin.Default() // includes Logger and Recovery middleware

    r.GET("/ping", func(c *gin.Context) {
        c.JSON(http.StatusOK, gin.H{"message": "pong"})

    r.Run(":8080") // blocks, listens on :8080

```

`gin.Context` is the core type — it wraps the request and response:

```go
// Reading request data

```

## 2.4 Binding & Validation

Gin can bind request bodies, query strings, and path params directly
into structs and validate them using struct tags.

```go
go get github.com/go-playground/validator/v10
```

```go
// Request structs with binding tags
type CreateBookRequest struct {
    Title       string  `json:"title"       binding:"required,min=1,max=200"`
    Author      string  `json:"author"      binding:"required"`
    Price       float64 `json:"price"       binding:"required,gt=0"`
    Stock       int     `json:"stock"       binding:"gte=0"`
    Year        int     `json:"year"        binding:"gte=1000,lte=2100"`
    Description string  `json:"description" binding:"max=2000"`
}

type ListBooksQuery struct {
    Page     int     `form:"page"      binding:"gte=0"`
    Limit    int     `form:"limit"     binding:"gte=0,lte=100"`
    Search   string  `form:"q"`
    Author   string  `form:"author"`
    MinPrice float64 `form:"min_price" binding:"gte=0"`
    MaxPrice float64 `form:"max_price" binding:"gte=0"`
}

// In handler
func (h *BookHandler) Create(c *gin.Context) {
    var req CreateBookRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
        return
    }
    // req is validated — Title is non-empty, Price > 0, etc.
}

func (h *BookHandler) List(c *gin.Context) {
    var query ListBooksQuery
    if err := c.ShouldBindQuery(&query); err != nil {
        c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
        return
    }
    // query.Page, query.Limit etc. are parsed and validated
}
```

### Custom Validation Error Messages

```go
import "github.com/go-playground/validator/v10"

func formatValidationErrors(err error) []string {
    var ve validator.ValidationErrors
    if !errors.As(err, &ve) {
        return []string{err.Error()}
    }

    msgs := make([]string, len(ve))
    for i, e := range ve {
        switch e.Tag() {
        case "required":
            msgs[i] = fmt.Sprintf("%s is required", e.Field())
        case "min":
            msgs[i] = fmt.Sprintf("%s must be at least %s characters", e.Field(), e.Param())
        case "gt":
            msgs[i] = fmt.Sprintf("%s must be greater than %s", e.Field(), e.Param())
        default:
            msgs[i] = fmt.Sprintf("%s failed validation: %s", e.Field(), e.Tag())
        }
    }
    return msgs
}

if err := c.ShouldBindJSON(&req); err != nil {
    c.JSON(http.StatusBadRequest, gin.H{
        "error":  "validation failed",
        "errors": formatValidationErrors(err),
```

```go
// Middleware function signature
func AuthMiddleware(jwtSecret string) gin.HandlerFunc {
    return func(c *gin.Context) {
        token := c.GetHeader("Authorization")
        if token == "" {
            c.AbortWithStatusJSON(http.StatusUnauthorized, gin.H{"error": "missing token"})
            return // c.Abort stops the chain — c.Next() would continue it
        }

        claims, err := jwt.Validate(strings.TrimPrefix(token, "Bearer "), jwtSecret)
        if err != nil {
            c.AbortWithStatusJSON(http.StatusUnauthorized, gin.H{"error": "invalid token"})
            return
        }

        // Store values in context for downstream handlers
        c.Set("userID", claims.UserID)
        c.Set("userRole", claims.Role)

        c.Next() // call the next handler/middleware
    }
}

// Rate limiter middleware
func RateLimiter(rps int) gin.HandlerFunc {
    limiter := rate.NewLimiter(rate.Limit(rps), rps)
    return func(c *gin.Context) {
        if !limiter.Allow() {
            c.AbortWithStatusJSON(http.StatusTooManyRequests, gin.H{
                "error": "rate limit exceeded",
            })
            return
        }
        c.Next()
    }
}

// Request ID middleware
func RequestID() gin.HandlerFunc {
    return func(c *gin.Context) {
        id := uuid.New().String()
        c.Set("requestID", id)
        c.Header("X-Request-ID", id)
        c.Next()
    }
}

// Get values set by middleware
func (h *Handler) myHandler(c *gin.Context) {
    userID, _ := c.Get("userID")
```

# Week 3 — Redis: Caching & Sessions

## 3.1 What Redis Is For

Redis is an in-memory key-value store. You use it when you need:
- **Caching** — avoid hitting the database for frequently read data
- **Sessions** — store user session data without database queries
- **Rate limiting** — count requests per IP per time window
- **Queues** — push/pop jobs for background workers (next week)
- **Pub/Sub** — real-time event broadcasting

**"In-memory" means:** Redis stores all data in RAM, which makes it extremely fast (sub-millisecond reads), but data can be lost if Redis crashes or restarts. Redis supports persistence mechanisms (RDB snapshots, AOF logs) to mitigate this, but for caching this is usually acceptable — the PostgreSQL database is your source of truth. **Never use Redis as your only data store for critical data** (orders, payments, user accounts).

## 3.5 Cache-Aside Pattern

Cache-aside (also called lazy loading) is the most common caching pattern:

```
Read path:
  1. Check cache
  2. Cache hit → return cached value
  3. Cache miss → query database → store in cache → return value

Write path:
  1. Write to database
  2. Delete (or update) cache entry
```

```go
// This is already implemented in usecase/book.go — here is the full pattern:
func (uc *BookUseCase) GetByID(ctx context.Context, id int) (*domain.Book, error) {
    key := fmt.Sprintf("book:%d", id)

    // 1. Try cache
    book, err := uc.cache.Get(ctx, key)
    if err == nil {
        return book, nil // cache hit
    }
    if !errors.Is(err, redis.ErrCacheMiss) {
        // Log cache error but don't fail the request
        slog.Warn("cache get error", "key", key, "err", err)
    }

    // 2. Cache miss — hit the database
    book, err = uc.repo.GetByID(ctx, id)
    if err != nil {
        return nil, err
    }

    // 3. Populate cache — if this fails, don't fail the request
    if err := uc.cache.Set(ctx, key, book); err != nil {
        slog.Warn("cache set error", "key", key, "err", err)
    }

    return book, nil
}
```

**Important:** Cache errors should not fail requests. If Redis is down, fall back to the database.

## 3.6 Redis for Rate Limiting

The sliding window counter pattern using Redis:

```go
// pkg/ratelimit/ratelimit.go
package ratelimit

import (
    "context"
    "fmt"
    "time"

    "github.com/redis/go-redis/v9"
)

type Limiter struct {
    client *redis.Client
    limit  int           // max requests
    window time.Duration // per window
}

func New(client *redis.Client, limit int, window time.Duration) *Limiter {
    return &Limiter{client: client, limit: limit, window: window}
}

// Allow returns true if the request is allowed, false if rate limited.
// key is typically the client IP or user ID.
func (l *Limiter) Allow(ctx context.Context, key string) (bool, int, error) {
    rKey := fmt.Sprintf("rate:%s", key)
    now := time.Now().UnixMilli()
    windowStart := now - l.window.Milliseconds()

    // Use a pipeline to execute multiple commands atomically
    pipe := l.client.Pipeline()

    // Remove old requests outside the window
    pipe.ZRemRangeByScore(ctx, rKey, "0", fmt.Sprintf("%d", windowStart))

    // Count current requests in window
    countCmd := pipe.ZCard(ctx, rKey)

    // Add this request
    pipe.ZAdd(ctx, rKey, redis.Z{Score: float64(now), Member: now})

    // Reset TTL
    pipe.Expire(ctx, rKey, l.window)

    if _, err := pipe.Exec(ctx); err != nil {
        return true, 0, err // fail open
    }

    count := int(countCmd.Val())
    remaining := l.limit - count - 1
    if remaining < 0 {
        remaining = 0
    }

    return count < l.limit, remaining, nil
}

// Gin middleware using the limiter
func GinMiddleware(limiter *Limiter) gin.HandlerFunc {
    return func(c *gin.Context) {
        key := c.ClientIP()

        allowed, remaining, err := limiter.Allow(c.Request.Context(), key)
        if err != nil {
            c.Next() // fail open on Redis error
            return
        }

        c.Header("X-RateLimit-Remaining", strconv.Itoa(remaining))

        if !allowed {
            c.AbortWithStatusJSON(http.StatusTooManyRequests, gin.H{
                "error": "too many requests",
            })
            return
        }

        c.Next()
    }
}
```

# Week 4 — Background Jobs & Workers

## 4.1 Why Background Jobs

Some work should not happen during the HTTP request:
- Sending a welcome email after registration
- Generating a PDF invoice after an order
- Processing a payment (retry-able)
- Resizing an uploaded image
- Sending notifications

Doing these in the handler makes the user wait and risks timeout errors.
Background jobs let you return `202 Accepted` immediately and process later.

**The trade-off:** Moving work to background jobs means the user gets a fast response, but the work might fail silently. You need:
- **Retry logic** — jobs can fail transiently (network blip, service down). Retry with exponential backoff.
- **Dead-letter queues** — after max retries, move failed jobs somewhere for investigation, don't lose them.
- **Monitoring** — you need to detect stuck or growing queues. If your email queue has 10,000 pending jobs, something is wrong.
- **Eventual consistency** — the user sees "order confirmed" immediately, but the confirmation email arrives minutes later. Design your UX around this.

---

## 4.2 Simple Goroutine-Based Worker Pool

For jobs that can be lost on restart (non-critical tasks like logging):

```go
// pkg/worker/pool.go
package worker

import (
    "context"
    "log/slog"
    "sync"
)

type Job func(ctx context.Context) error

type Pool struct {
    jobs    chan Job
    workers int
    wg      sync.WaitGroup
}

func NewPool(workers, queueSize int) *Pool {
    return &Pool{
        jobs:    make(chan Job, queueSize),
        workers: workers,
    }
}

func (p *Pool) Start(ctx context.Context) {
    for i := 0; i < p.workers; i++ {
        p.wg.Add(1)
        go func(id int) {
            defer p.wg.Done()
            for {
                select {
                case job, ok := <-p.jobs:
                    if !ok {
                        return // channel closed
                    }
                    if err := job(ctx); err != nil {
                        slog.Error("job failed", "worker", id, "err", err)
                    }
                case <-ctx.Done():
                    return
                }
            }
        }(i)
    }
}

// Submit adds a job to the queue. Returns false if the queue is full.
func (p *Pool) Submit(job Job) bool {
    select {
    case p.jobs <- job:
        return true
    default:
        return false // queue full — drop or handle
    }
}

func (p *Pool) Stop() {
    close(p.jobs)
    p.wg.Wait()
}
```

Usage:

```go
pool := worker.NewPool(4, 100) // 4 workers, queue of 100 jobs
pool.Start(ctx)
defer pool.Stop()

// In an HTTP handler — fire and forget
pool.Submit(func(ctx context.Context) error {
    return emailService.SendWelcome(ctx, user.Email)
})
```

## 4.3 Redis-Backed Job Queue

For jobs that must survive restarts (critical: payments, invoices):

```go
// pkg/queue/queue.go
package queue

import (
    "context"
    "encoding/json"
    "fmt"
    "time"

    "github.com/redis/go-redis/v9"
)

type Job struct {
    ID        string          `json:"id"`
    Type      string          `json:"type"`
    Payload   json.RawMessage `json:"payload"`
    Attempts  int             `json:"attempts"`
    CreatedAt time.Time       `json:"created_at"`
}

type Queue struct {
    client *redis.Client
    name   string
}

func New(client *redis.Client, name string) *Queue {
    return &Queue{client: client, name: name}
}

func (q *Queue) key() string {
    return fmt.Sprintf("queue:%s", q.name)
}

func (q *Queue) deadKey() string {
    return fmt.Sprintf("queue:%s:dead", q.name)
}

// Enqueue adds a job to the right end of the list
func (q *Queue) Enqueue(ctx context.Context, jobType string, payload any) error {
    data, err := json.Marshal(payload)
    if err != nil {
        return err
    }

    job := Job{
        ID:        uuid.New().String(),
        Type:      jobType,
        Payload:   data,
        CreatedAt: time.Now(),
    }

    b, err := json.Marshal(job)
    if err != nil {
        return err
    }

    return q.client.RPush(ctx, q.key(), b).Err()
}

// Dequeue blocks until a job is available (timeout: 5s)
func (q *Queue) Dequeue(ctx context.Context) (*Job, error) {
    // BLPOP blocks waiting for a message — efficient, no busy-polling
    result, err := q.client.BLPop(ctx, 5*time.Second, q.key()).Result()
    if err == redis.Nil {
        return nil, nil // timeout — no jobs available
    }
    if err != nil {
        return nil, fmt.Errorf("dequeue: %w", err)
    }

    var job Job
    if err := json.Unmarshal([]byte(result[1]), &job); err != nil {
        return nil, fmt.Errorf("unmarshal job: %w", err)
    }

    return &job, nil
}

// MoveToDead sends a failed job to the dead letter queue
func (q *Queue) MoveToDead(ctx context.Context, job *Job) error {
    b, _ := json.Marshal(job)
    return q.client.RPush(ctx, q.deadKey(), b).Err()
}

// Size returns the number of pending jobs
func (q *Queue) Size(ctx context.Context) (int64, error) {
    return q.client.LLen(ctx, q.key()).Result()
}
```

## 4.4 Job Dispatcher & Handlers

```go
// pkg/queue/dispatcher.go
package queue

import (
    "context"
    "encoding/json"
    "log/slog"
)

// Handler processes a specific job type
type Handler func(ctx context.Context, payload json.RawMessage) error

type Dispatcher struct {
    queue    *Queue
    handlers map[string]Handler
    maxRetry int
}

func NewDispatcher(queue *Queue, maxRetry int) *Dispatcher {
    return &Dispatcher{
        queue:    queue,
        handlers: make(map[string]Handler),
        maxRetry: maxRetry,
    }
}

func (d *Dispatcher) Register(jobType string, handler Handler) {
    d.handlers[jobType] = handler
}

func (d *Dispatcher) Run(ctx context.Context) {
    slog.Info("dispatcher started")
    for {
        select {
        case <-ctx.Done():
            slog.Info("dispatcher stopping")
            return
        default:
        }

        job, err := d.queue.Dequeue(ctx)
        if err != nil {
            slog.Error("dequeue error", "err", err)
            time.Sleep(time.Second) // back off on error
            continue
        }
        if job == nil {
            continue // timeout — no jobs
        }

        d.process(ctx, job)
    }
}

func (d *Dispatcher) process(ctx context.Context, job *Job) {
    handler, ok := d.handlers[job.Type]
    if !ok {
        slog.Error("no handler for job type", "type", job.Type, "id", job.ID)
        d.queue.MoveToDead(ctx, job)
        return
    }

    job.Attempts++
    err := handler(ctx, job.Payload)
    if err == nil {
        slog.Info("job completed", "type", job.Type, "id", job.ID, "attempts", job.Attempts)
        return
    }

    slog.Warn("job failed", "type", job.Type, "id", job.ID, "attempts", job.Attempts, "err", err)

    if job.Attempts >= d.maxRetry {
        slog.Error("job exceeded max retries — moving to dead letter", "id", job.ID)
        d.queue.MoveToDead(ctx, job)
        return
    }

    // Re-queue with exponential backoff
    delay := time.Duration(job.Attempts) * 2 * time.Second
    time.AfterFunc(delay, func() {
        b, _ := json.Marshal(job)
        d.queue.client.RPush(ctx, d.queue.key(), b)
    })
}
```

## 4.6 Scheduled Jobs — Cron-style Tasks

```go
// pkg/scheduler/scheduler.go
package scheduler

import (
    "context"
    "log/slog"
    "time"
)

type Task struct {
    Name     string
    Interval time.Duration
    Fn       func(ctx context.Context) error
}

type Scheduler struct {
    tasks []Task
}

func New() *Scheduler {
    return &Scheduler{}
}

func (s *Scheduler) Add(name string, interval time.Duration, fn func(context.Context) error) {
    s.tasks = append(s.tasks, Task{Name: name, Interval: interval, Fn: fn})
}

func (s *Scheduler) Run(ctx context.Context) {
    for _, task := range s.tasks {
        go s.runTask(ctx, task)
    }
    <-ctx.Done()
}

func (s *Scheduler) runTask(ctx context.Context, t Task) {
    ticker := time.NewTicker(t.Interval)
    defer ticker.Stop()

    slog.Info("scheduled task registered", "name", t.Name, "interval", t.Interval)

    for {
        select {
        case <-ticker.C:
            slog.Info("running scheduled task", "name", t.Name)
            if err := t.Fn(ctx); err != nil {
                slog.Error("scheduled task failed", "name", t.Name, "err", err)
            }
        case <-ctx.Done():
            return
        }
    }
}
```

# Week 5 — Docker & Deployment

## 5.1 Why Docker

Docker packages your app and all its dependencies into a container that runs
identically on every machine. No more "it works on my laptop."

**Key concepts:**
- A **Docker image** is a snapshot of a filesystem plus metadata (what command to run, what ports to expose, what environment variables to set). It is built once and is immutable.
- A **container** is a running instance of an image, isolated from the host using Linux namespaces and cgroups. You can run multiple containers from the same image.
- Unlike virtual machines, containers **share the host kernel** and start in milliseconds, not minutes. They are much lighter (MBs, not GBs).

For Go specifically, Docker is elegant: Go compiles to a single static binary
with no runtime dependencies. The final image can be tiny (< 20MB).

---

## 5.2 Writing a Dockerfile

```dockerfile
# Dockerfile

# ── Stage 1: Build ──────────────────────────────────────────────────
FROM golang:1.22-alpine AS builder

# Install git and ca-certificates (needed for go get)
RUN apk add --no-cache git ca-certificates tzdata

WORKDIR /app

# Copy dependency files first — cached unless go.mod/go.sum change
COPY go.mod go.sum ./
RUN go mod download

# Copy source code
COPY . .

# Build a statically linked binary — no external dependencies
RUN CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build \
    -ldflags="-w -s" \
    -o /app/server \
    ./cmd/api

# ── Stage 2: Final image ─────────────────────────────────────────────
FROM scratch
# 'scratch' is an empty image — your binary is all that runs

# Copy only what we need from the builder
COPY --from=builder /etc/ssl/certs/ca-certificates.crt /etc/ssl/certs/
COPY --from=builder /usr/share/zoneinfo /usr/share/zoneinfo
COPY --from=builder /app/server /server
COPY --from=builder /app/migrations /migrations

# Expose the port
EXPOSE 8080

# Run as non-root (good security practice)
USER 65534:65534

ENTRYPOINT ["/server"]
```

Build and run:

```bash
docker build -t bookstore:latest .
docker run -p 8080:8080 \
  -e DATABASE_URL="postgres://..." \
  -e REDIS_URL="redis://..." \
  -e JWT_SECRET="..." \
  bookstore:latest
```

**Multi-stage builds** are the key. The `builder` stage has the Go toolchain (~300MB).
The final `scratch` stage has only your binary (~10MB). This means:
- Faster image pulls
- Smaller attack surface
- No shell, no package manager to exploit

## 5.7 Health Checks & Readiness

A proper health check tells load balancers and orchestrators if your service is ready:

```go
type HealthStatus struct {
    Status   string            `json:"status"`
    Services map[string]string `json:"services"`
}

func HealthHandler(db *sql.DB, redisClient *redis.Client) gin.HandlerFunc {
    return func(c *gin.Context) {
        status := HealthStatus{
            Status:   "ok",
            Services: make(map[string]string),
        }

        // Check database
        ctx, cancel := context.WithTimeout(c.Request.Context(), 2*time.Second)
        defer cancel()

        if err := db.PingContext(ctx); err != nil {
            status.Status = "degraded"
            status.Services["database"] = "unhealthy: " + err.Error()
        } else {
            status.Services["database"] = "healthy"
        }

        // Check Redis
        if err := redisClient.Ping(ctx).Err(); err != nil {
            status.Status = "degraded"
            status.Services["redis"] = "unhealthy: " + err.Error()
        } else {
            status.Services["redis"] = "healthy"
        }

        httpStatus := http.StatusOK
        if status.Status != "ok" {
            httpStatus = http.StatusServiceUnavailable
        }

        c.JSON(httpStatus, status)
    }
}
```

# Week 6 — Advanced Testing & Capstone

## 6.1 Testing Strategies in Go

There are three levels of tests you should write:

**Unit tests** — test a single function or method in isolation. No database, no network.
Fast (milliseconds). Use fakes/mocks for dependencies.

**Integration tests** — test a component with its real dependencies (real database, real Redis).
Slower (seconds). Use a test database, tear down after each test.

**End-to-end tests** — test the full HTTP stack. Spin up the server, send real HTTP requests.
Slowest (tens of seconds). Test happy paths and critical error cases.

**The testing pyramid:** Aim for many unit tests, fewer integration tests, and fewest e2e tests. Unit tests catch logic bugs fast and are cheap to run. Integration tests catch wiring and SQL bugs. E2e tests catch routing and middleware bugs. If a bug can be caught at a lower level, write the test there — it will be faster and more stable.

## 6.3 Fake Implementations vs Mocks

There are two main approaches to replacing real dependencies in tests:

**Mocks** are typically generated by tools like `gomock` or `mockery`. They record which methods were called, with what arguments, and let you assert on those calls. Mocks are brittle — they test *implementation details* rather than behavior. If you refactor the internal code to call methods in a different order, mock tests break even if the result is correct.

**Fakes** are hand-written implementations of interfaces with simple behavior.
They test *behavior*: "given this input, does the use case produce the right output?" Fakes don't care about the order of internal calls.

**Prefer fakes.** Use mocks only when you need to verify a specific interaction (e.g., "was the email service called exactly once with this address?"). For most tests, fakes give you better coverage with less maintenance.

```go
// testutil/fake_book_repo.go
package testutil

import (
    "context"
    "sync"

    "github.com/yourname/bookstore/internal/domain"
)

type FakeBookRepo struct {
    mu           sync.RWMutex
    books        map[int]*domain.Book
    nextID       int
    createCalled bool
    createErr    error
    getErr       error
}

func NewFakeBookRepo() *FakeBookRepo {
    return &FakeBookRepo{books: make(map[int]*domain.Book), nextID: 1}
}

func (r *FakeBookRepo) WithCreateError(err error) *FakeBookRepo {
    r.createErr = err
    return r
}

func (r *FakeBookRepo) Create(ctx context.Context, b *domain.Book) (*domain.Book, error) {
    r.createCalled = true
    if r.createErr != nil {
        return nil, r.createErr
    }
    r.mu.Lock()
    defer r.mu.Unlock()
    b.ID = r.nextID
    r.nextID++
    copy := *b
    r.books[b.ID] = &copy
    return b, nil
}

func (r *FakeBookRepo) GetByID(ctx context.Context, id int) (*domain.Book, error) {
    if r.getErr != nil {
        return nil, r.getErr
    }
    r.mu.RLock()
    defer r.mu.RUnlock()
    b, ok := r.books[id]
    if !ok {
        return nil, domain.ErrBookNotFound
    }
    return b, nil
}

// Implement the rest of the interface with sensible no-op or basic behavior
func (r *FakeBookRepo) List(ctx context.Context, f domain.BookFilter) ([]domain.Book, int, error) {
    r.mu.RLock()
    defer r.mu.RUnlock()
    var books []domain.Book
    for _, b := range r.books {
        books = append(books, *b)
    }
    return books, len(books), nil
}

func (r *FakeBookRepo) Update(ctx context.Context, b *domain.Book) (*domain.Book, error) {
    r.mu.Lock()
    defer r.mu.Unlock()
    if _, ok := r.books[b.ID]; !ok {
        return nil, domain.ErrBookNotFound
    }
    r.books[b.ID] = b
    return b, nil
}

func (r *FakeBookRepo) Delete(ctx context.Context, id int) error {
    r.mu.Lock()
    defer r.mu.Unlock()
    delete(r.books, id)
    return nil
}

func (r *FakeBookRepo) UpdateStock(ctx context.Context, id, delta int) error {
    r.mu.Lock()
    defer r.mu.Unlock()
    if b, ok := r.books[id]; ok {
        b.Stock += delta
    }
    return nil
}
```

```go
        Title:  "The Go Programming Language",
        Author: "Donovan",
        Price:  39.99,
        Stock:  10,
    }
    created, err := repo.Create(ctx, book)
    if err != nil {
        t.Fatalf("Create: %v", err)
    }
    if created.ID == 0 {
        t.Fatal("expected non-zero ID after create")
    }

    // Get
    got, err := repo.GetByID(ctx, created.ID)
    if err != nil {
        t.Fatalf("GetByID: %v", err)
    }
    if got.Title != book.Title {
        t.Errorf("Title: got %q; want %q", got.Title, book.Title)
    }
}
```
