# Minggu 1 — Setup, Sintaks & Tipe Dasar

## 1.1 Instalasi Go & Program Pertamamu

Download Go dari https://go.dev/dl dan instal. Lalu verifikasi:

```bash
go version
# go version go1.22.0 linux/amd64
# (your version and OS may differ — e.g., darwin/arm64 on macOS)
```

Siapkan VS Code dengan ekstensi Go resmi (`golang.go`). Ekstensi ini memberikan autocompletion, formatting otomatis saat save, dan petunjuk error secara inline.

### Perintah Go yang Penting

Sebelum mulai coding, ketahui perintah-perintah ini — kamu akan menggunakannya setiap hari:

```bash
go fmt ./...       # auto-formats all Go files (run this often)
go vet ./...       # finds common mistakes the compiler misses
go run main.go     # compile and run in one step
go build -o app    # compile to a standalone binary (no runtime needed)
go test ./...      # run all tests
go mod tidy        # clean up unused dependencies
```

### Program Pertamamu

Buat sebuah folder, inisialisasi module, dan tulis hello world:

```bash
mkdir learn-go && cd learn-go
go mod init learn-go
```

`go mod init` membuat file `go.mod`, yang mendefinisikan **module** kamu — sebuah kumpulan package Go yang saling terkait dengan nama dan pelacakan dependensi. Anggap saja seperti `package.json` di Node.js atau `requirements.txt` di Python, tapi sudah built-in ke dalam bahasa. Setiap proyek Go memerlukan sebuah module.

Buat `main.go`:

```go
package main

import "fmt"

func main() {
    fmt.Println("Hello, World!")
}
```

Jalankan:

```bash
go run main.go
# Hello, World!
```

Build menjadi binary:

```bash
go build -o myapp main.go
./myapp
```

**Hal-hal penting yang perlu diperhatikan:**
- Setiap file Go diawali dengan `package name`
- `package main` itu spesial — menandakan titik masuk (entry point) sebuah program
- `func main()` adalah tempat eksekusi dimulai
- `import` mengimpor package. `fmt` adalah package untuk formatting/printing
- `go build -o myapp` mengompilasi kode kamu menjadi binary mandiri — tidak perlu runtime atau VM. Kamu bisa menyalin satu file itu ke mesin mana pun dengan OS/arsitektur yang sama dan langsung bisa dijalankan

## 1.2 Variabel & Tipe

Go bertipe statis (statically typed). Setiap variabel punya tipe, dan tipe tidak dikonversi secara otomatis.

### Mendeklarasikan Variabel

```go
// Long form — explicit type
var name string = "Alice"
var age int = 30
var active bool = true

// Short form — type inferred (only inside functions)
name := "Alice"
age := 30
active := true

// Multiple variables
var x, y int = 10, 20
a, b := "hello", "world"

// Block declaration
var (
    host string = "localhost"
    port int    = 8080
)
```

**Aturan:** Gunakan `:=` di dalam fungsi (hampir selalu). Gunakan `var` untuk variabel level package atau ketika kamu ingin mendeklarasikan tanpa langsung mengisi nilai.

### Zero Value

Di Go, variabel selalu punya nilai — meskipun kamu tidak mengisinya. Nilai default-nya adalah **zero value** dari tipe tersebut.

```go
var i int     // 0
var f float64 // 0.0
var b bool    // false
var s string  // ""  (empty string, not null)
var p *int    // nil
```

Ini disengaja. Go tidak punya `undefined` atau `null` untuk tipe dasar. Terima dan manfaatkan hal ini.

### Tipe Dasar

```go
// Integer types
var a int    = 42       // platform size (32 or 64 bit)
var b int8   = 127      // -128 to 127
var c int16  = 32767
var d int32  = 2147483647
var e int64  = 9223372036854775807
var u uint   = 42       // unsigned (no negatives)

// Float types
var f32 float32 = 3.14
var f64 float64 = 3.141592653589793  // prefer float64

// Boolean
var yes bool = true
var no  bool = false

// String
var greeting string = "Hello, Go!"

// Rune (a Unicode code point, alias for int32)
var letter rune = 'A'

// Byte (alias for uint8)
var by byte = 255
```

**`byte` vs `rune` — kapan menggunakan yang mana:**
- `byte` adalah alias untuk `uint8` (0–255). Gunakan untuk karakter ASCII dan data binary mentah.
- `rune` adalah alias untuk `int32`. Merepresentasikan satu Unicode code point — karakter apa pun dalam bahasa apa pun.
- String di Go secara internal adalah urutan byte, tapi ketika kamu iterasi dengan `range`, Go akan men-decode-nya menjadi rune secara otomatis:

```go
s := "Hello, 世界"
for i, r := range s {
    fmt.Printf("index %d → rune %c (bytes: %d)\n", i, r, utf8.RuneLen(r))
}
// index 0 → rune H (bytes: 1)
// index 7 → rune 世 (bytes: 3)   ← notice: index jumps from 6 to 7 because 世 is 3 bytes
// index 10 → rune 界 (bytes: 3)
```

**Aturan praktis:** Gunakan `byte` untuk ASCII/binary, `rune` untuk pemrosesan teks dengan karakter internasional.

### Konversi Tipe

Go TIDAK melakukan konversi implisit. Kamu harus mengonversi secara eksplisit:

```go
var i int = 42
var f float64 = float64(i)   // must convert
var u uint = uint(f)

// This would NOT compile:
// var f float64 = i  // error: cannot use i (type int) as type float64
```

### Konstanta

```go
const Pi = 3.14159
const MaxUsers = 1000
const AppName = "MyApp"

// Typed constant
const TypedPi float64 = 3.14159

// Block of constants
const (
    StatusOK    = 200
    StatusError = 500
)
```

### iota — Konstanta Auto-increment

```go
type Direction int

const (
    North Direction = iota // 0
    East                   // 1
    South                  // 2
    West                   // 3
)

// More creative iota usage
const (
    _           = iota // skip 0
    KB = 1 << (10 * iota) // 1024
    MB                    // 1048576
    GB                    // 1073741824
)
```

## 1.3 Alur Kontrol

### if / else

```go
age := 20

if age >= 18 {
    fmt.Println("adult")
} else if age >= 13 {
    fmt.Println("teenager")
} else {
    fmt.Println("child")
}
```

**`if` spesial di Go — init statement:**

```go
// The variable declared in the init statement is scoped to the if block
if n := 10; n > 5 {
    fmt.Println("n is greater than 5")
}
// n is not accessible here

// Very common pattern with errors:
if err := doSomething(); err != nil {
    fmt.Println("error:", err)
    return
}
```

### for — Satu-satunya Loop di Go

Go tidak punya `while`. Loop `for` menangani semuanya:

```go
// Classic C-style for
for i := 0; i < 5; i++ {
    fmt.Println(i)
}

// Like a while loop
n := 0
for n < 5 {
    n++
}

// Infinite loop
for {
    // runs forever
    break // use break to exit
}

// Range over a slice
fruits := []string{"apple", "banana", "cherry"}
for i, fruit := range fruits {
    fmt.Printf("%d: %s\n", i, fruit)
}

// Range — ignore index
for _, fruit := range fruits {
    fmt.Println(fruit)
}

// Range over a map
scores := map[string]int{"Alice": 90, "Bob": 85}
for name, score := range scores {
    fmt.Printf("%s: %d\n", name, score)
}

// Range over a string (gives runes, not bytes)
for i, ch := range "hello" {
    fmt.Printf("%d: %c\n", i, ch)
}
```

### switch

```go
day := "Monday"

switch day {
case "Monday", "Tuesday", "Wednesday", "Thursday", "Friday":
    fmt.Println("Weekday")
case "Saturday", "Sunday":
    fmt.Println("Weekend")
default:
    fmt.Println("Unknown")
}

// Switch with no expression (like if/else if chains)
score := 75
switch {
case score >= 90:
    fmt.Println("A")
case score >= 80:
    fmt.Println("B")
case score >= 70:
    fmt.Println("C")
default:
    fmt.Println("F")
}
```

**Catatan:** Berbeda dengan C, case di switch Go TIDAK fall through secara default. Gunakan keyword `fallthrough` secara eksplisit jika kamu membutuhkannya (jarang).

### defer

`defer` menjadwalkan pemanggilan fungsi untuk dijalankan tepat sebelum fungsi yang melingkupinya return. Panggilan yang di-defer berjalan dalam urutan LIFO (last in, first out).

```go
func readFile() {
    file, err := os.Open("data.txt")
    if err != nil {
        return
    }
    defer file.Close() // will run when readFile() exits, no matter what

    // work with file...
}

// Deferred calls run in LIFO
func counting() {
    for i := 0; i < 3; i++ {
        defer fmt.Println(i)
    }
    // Output: 2, 1, 0
}
```

**Penggunaan umum:** menutup file, melepas lock, menutup koneksi database.

**Jebakan #1 — Argumen dievaluasi langsung:**

```go
func main() {
    x := 10
    defer fmt.Println("deferred x =", x)  // captures x = 10 NOW
    x = 20
    fmt.Println("current x =", x)
}
// Output:
// current x = 20
// deferred x = 10   ← NOT 20! The argument was evaluated at the defer statement
```

**Jebakan #2 — Hindari defer di dalam loop:**

```go
// BAD — opens many files but only closes them all when the function exits
for _, name := range files {
    f, _ := os.Open(name)
    defer f.Close()  // these all pile up until the function returns!
}

// GOOD — wrap in a helper function so each defer runs immediately
for _, name := range files {
    processFile(name)
}

func processFile(name string) {
    f, _ := os.Open(name)
    defer f.Close()  // closes when processFile returns (each iteration)
    // work with f...
}
```

## 1.4 Array, Slice & Map

### Array

Array punya ukuran tetap. Ukurannya merupakan bagian dari tipe. Jarang digunakan langsung di Go.

```go
var arr [5]int          // [0 0 0 0 0]
arr[0] = 10

primes := [5]int{2, 3, 5, 7, 11}
auto := [...]int{1, 2, 3}  // compiler counts the size

fmt.Println(len(primes))   // 5

// Arrays are value types — copying creates a full copy
a := [3]int{1, 2, 3}
b := a   // b is a separate copy
b[0] = 99
fmt.Println(a[0]) // still 1
```

### Slice

Slice adalah pekerja keras di Go. Sebuah slice adalah tampilan (view) ke array yang mendasarinya — punya panjang (length) dan kapasitas (capacity).

```go
// Creating slices
s := []int{1, 2, 3, 4, 5}
empty := []int{}
withMake := make([]int, 5)       // length 5, cap 5, all zeros
withCap  := make([]int, 3, 10)   // length 3, cap 10

// Accessing and modifying
fmt.Println(s[0])  // 1
s[0] = 99

// Slicing a slice [low:high] — includes low, excludes high
fmt.Println(s[1:3])  // [2 3]
fmt.Println(s[:2])   // [1 2]  (from beginning)
fmt.Println(s[2:])   // [3 4 5] (to end)

// Appending — returns a new slice (may reallocate)
s = append(s, 6, 7)
other := []int{8, 9}
s = append(s, other...)  // spread another slice

// Length vs Capacity
s2 := make([]int, 3, 5)
fmt.Println(len(s2), cap(s2))  // 3 5

// Copy
src := []int{1, 2, 3}
dst := make([]int, len(src))
copy(dst, src)
```

**Model mentalnya:** Sebuah slice adalah struct dengan tiga field: pointer ke array, panjang (length), dan kapasitas (capacity). Ketika kamu append melebihi kapasitas, Go mengalokasikan array baru yang lebih besar dan menyalin datanya.

```go
// Iterating slices
nums := []int{10, 20, 30}
for i, v := range nums {
    fmt.Printf("index %d, value %d\n", i, v)
}

// 2D slice
matrix := [][]int{
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9},
}
fmt.Println(matrix[1][2]) // 6
```

**Jebakan — Slice berbagi backing array yang sama:**

```go
original := []int{1, 2, 3, 4, 5}
sub := original[1:3]  // sub = [2 3]

sub[0] = 99
fmt.Println(original)  // [1 99 3 4 5] — original was modified!
```

Ketika kamu memotong (slice) sebuah slice, keduanya menunjuk ke array dasar yang sama. Untuk mendapatkan salinan independen, gunakan `copy` atau ekspresi slice lengkap `original[1:3:3]` (membatasi kapasitas).

**nil slice vs empty slice:**

```go
var nilSlice []int          // nil — the idiomatic zero value
emptySlice := []int{}       // non-nil, but length 0

fmt.Println(nilSlice == nil)    // true
fmt.Println(emptySlice == nil)  // false
fmt.Println(len(nilSlice))      // 0 — both work the same with len/append
```

Gunakan `var s []int` (nil) sebagai default. Baik nil maupun empty slice berfungsi sama dengan `len`, `append`, dan `range`.

> **Sejak Go 1.21:** Package `slices` di standard library menyediakan helper berguna seperti `slices.Contains`, `slices.Sort`, `slices.Index`, dan lainnya. Impor dengan `import "slices"`.

### Map

Map adalah hash table bawaan Go.

```go
// Creating maps
ages := map[string]int{
    "Alice": 30,
    "Bob":   25,
}

empty := make(map[string]int)

// Reading — missing key returns zero value (0 for int)
fmt.Println(ages["Alice"])   // 30
fmt.Println(ages["Charlie"]) // 0 — key doesn't exist, no panic

// Check if key exists — comma-ok idiom
val, ok := ages["Charlie"]
if !ok {
    fmt.Println("Charlie not found")
}

// Writing
ages["Charlie"] = 35
ages["Alice"] = 31   // update

// Deleting
delete(ages, "Bob")

// Iterating — order is random, don't rely on it
for name, age := range ages {
    fmt.Printf("%s is %d\n", name, age)
}

// Length
fmt.Println(len(ages))
```

**Jebakan #1 — Menulis ke nil map akan panic:**

```go
var m map[string]int  // m is nil
fmt.Println(m["key"]) // 0 — reading a nil map is fine
m["key"] = 1          // PANIC: assignment to entry in nil map
```

Selalu inisialisasi map dengan `make(map[string]int)` atau literal sebelum menulis ke dalamnya.

**Jebakan #2 — Map TIDAK aman untuk akses konkuren.** Jika beberapa goroutine membaca dan menulis ke map yang sama secara bersamaan, program kamu akan crash. Kamu butuh `sync.Mutex` atau `sync.Map` (dibahas di Minggu 5).

> **Sejak Go 1.21:** Package `maps` menyediakan helper seperti `maps.Keys`, `maps.Values`, `maps.Clone`, dan `maps.Equal`. Impor dengan `import "maps"`.

## 1.5 Fungsi

Fungsi adalah warga kelas satu (first-class citizen) di Go.

### Fungsi Dasar

```go
// Simple function
func add(a int, b int) int {
    return a + b
}

// Shorthand when params share a type
func add(a, b int) int {
    return a + b
}

// No return value
func greet(name string) {
    fmt.Println("Hello,", name)
}
```

### Multiple Return Value

Ini adalah salah satu fitur paling khas dari Go.

```go
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, fmt.Errorf("cannot divide by zero")
    }
    return a / b, nil
}

// Calling it
result, err := divide(10, 3)
if err != nil {
    fmt.Println("Error:", err)
    return
}
fmt.Println(result) // 3.3333...

// Ignore a return value with _
result, _ = divide(10, 3)  // note: = not := because result is already declared
```

### Named Return Value

```go
func minMax(nums []int) (min, max int) {
    min, max = nums[0], nums[0]
    for _, n := range nums[1:] {
        if n < min { min = n }
        if n > max { max = n }
    }
    return // "naked" return — returns named values
}
```

Gunakan secukupnya. Named return membantu di fungsi pendek tapi mengurangi keterbacaan di fungsi yang lebih panjang.

### Fungsi Variadic

```go
func sum(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}

sum(1, 2, 3)        // 6
sum(1, 2, 3, 4, 5)  // 15

// Spread a slice
nums := []int{1, 2, 3}
sum(nums...)  // spread the slice
```

### Fungsi sebagai Nilai

```go
// Assign a function to a variable
double := func(n int) int {
    return n * 2
}
fmt.Println(double(5)) // 10

// Pass a function as an argument
func apply(nums []int, fn func(int) int) []int {
    result := make([]int, len(nums))
    for i, n := range nums {
        result[i] = fn(n)
    }
    return result
}

doubled := apply([]int{1, 2, 3}, func(n int) int { return n * 2 })
// [2 4 6]
```

### Closure

Closure menangkap variabel dari scope di sekitarnya.

```go
func counter() func() int {
    count := 0
    return func() int {
        count++
        return count
    }
}

c := counter()
fmt.Println(c()) // 1
fmt.Println(c()) // 2
fmt.Println(c()) // 3

// Each call to counter() creates an independent counter
c2 := counter()
fmt.Println(c2()) // 1 — independent from c
```

# Minggu 2 — Struct, Method & Pointer

## 2.1 Struct

Struct adalah cara Go mengelompokkan data yang saling terkait. Go tidak punya class — struct ditambah method mengisi peran tersebut.

```go
// Defining a struct
type Person struct {
    Name string
    Age  int
    Email string
}

// Creating a struct
p1 := Person{Name: "Alice", Age: 30, Email: "alice@example.com"}
p2 := Person{"Bob", 25, "bob@example.com"}  // positional (avoid this — fragile)

// Zero value struct
var p3 Person  // Name: "", Age: 0, Email: ""

// Accessing fields
fmt.Println(p1.Name)   // Alice
p1.Age = 31            // modify

// Struct as function argument
func greet(p Person) {
    fmt.Printf("Hello, %s!\n", p.Name)
}
```

### Struct Tag

Struct tag adalah string metadata yang dilampirkan ke field struct. Tag digunakan oleh package seperti `encoding/json` untuk mengontrol bagaimana field di-serialize.

```go
type User struct {
    ID        int    `json:"id"`
    FirstName string `json:"first_name"`
    Email     string `json:"email"`
    Password  string `json:"-"`              // "-" means never include in JSON
    Age       int    `json:"age,omitempty"`   // omit if zero value
}

u := User{ID: 1, FirstName: "Alice", Email: "alice@example.com", Password: "secret"}
data, _ := json.Marshal(u)
fmt.Println(string(data))
// {"id":1,"first_name":"Alice","email":"alice@example.com","age":0}
// Note: Password is excluded, age is 0 but still shown (omitempty would omit it if set)
```

Tag juga digunakan oleh library database (`db:"column_name"`), library validasi (`validate:"required"`), dan lainnya. Kita akan menggunakannya secara ekstensif di bab-bab selanjutnya.

### Konvensi Constructor — `NewXxx`

Go tidak punya constructor. Konvensinya adalah membuat fungsi bernama `NewXxx` yang mengembalikan struct yang sudah diinisialisasi:

```go
func NewPerson(name, email string, age int) Person {
    return Person{
        Name:  name,
        Age:   age,
        Email: email,
    }
}

// Usage
p := NewPerson("Alice", "alice@example.com", 30)
```

Ini sangat penting ketika struct punya field yang memerlukan validasi atau nilai default.

### Struct Literal

```go
type Point struct {
    X, Y float64
}

origin := Point{X: 0, Y: 0}
p := Point{X: 3.5, Y: -1.2}

// Anonymous struct — useful for quick grouping
config := struct {
    Host string
    Port int
}{
    Host: "localhost",
    Port: 8080,
}
```

### Membandingkan Struct

Struct bisa dibandingkan jika semua field-nya bertipe yang bisa dibandingkan:

```go
a := Point{1, 2}
b := Point{1, 2}
fmt.Println(a == b)  // true

// Structs with slices/maps are NOT comparable
type Team struct {
    Name    string
    Members []string  // slice — not comparable
}
// t1 == t2  // this would not compile
```

### Embedded Struct (Komposisi)

Go lebih mengutamakan komposisi daripada pewarisan (inheritance). Kamu menanamkan (embed) satu struct ke dalam struct lain:

```go
type Animal struct {
    Name string
}

func (a Animal) Speak() string {
    return a.Name + " makes a sound"
}

type Dog struct {
    Animal          // embedded — not a named field
    Breed string
}

d := Dog{
    Animal: Animal{Name: "Rex"},
    Breed:  "Labrador",
}

fmt.Println(d.Name)     // promoted field — access directly
fmt.Println(d.Speak())  // promoted method — access directly
fmt.Println(d.Breed)
```

Field dan method dari embedded struct **dipromosikan** ke struct luar. Ini bukan inheritance — ini adalah promosi.

## 2.2 Pointer

Pointer menyimpan alamat memori dari sebuah nilai. Go menggunakan pointer secara eksplisit, tidak seperti Java di mana referensi objek disembunyikan.

```go
x := 42
p := &x        // p is a *int, holds the address of x
fmt.Println(p)  // something like 0xc00001e090
fmt.Println(*p) // 42 — dereference to get the value

*p = 100       // change the value through the pointer
fmt.Println(x) // 100 — x is now 100
```

### Mengapa Pointer Penting

```go
// Without pointer — function gets a COPY
func doubleVal(n int) {
    n = n * 2  // only changes the local copy
}

// With pointer — function gets the real variable
func doublePtr(n *int) {
    *n = *n * 2  // changes the original
}

x := 5
doubleVal(x)
fmt.Println(x)  // still 5

doublePtr(&x)
fmt.Println(x)  // 10
```

### Pointer ke Struct

```go
type User struct {
    Name  string
    Score int
}

u := User{Name: "Alice", Score: 100}
p := &u

// Go auto-dereferences struct pointer fields
p.Score = 200       // same as (*p).Score = 200
fmt.Println(u.Score) // 200

// new() allocates a zeroed struct and returns a pointer
u2 := new(User)
u2.Name = "Bob"
```

### nil Pointer

```go
var p *int  // nil — doesn't point to anything

// Dereferencing nil PANICS
// fmt.Println(*p)  // panic: runtime error: invalid memory address

// Always check before dereferencing
if p != nil {
    fmt.Println(*p)
}
```

## 2.3 Method

Method adalah fungsi dengan **receiver** — struct (atau tipe) tempat ia dimiliki.

```go
type Rectangle struct {
    Width  float64
    Height float64
}

// Method with a VALUE receiver
func (r Rectangle) Area() float64 {
    return r.Width * r.Height
}

func (r Rectangle) Perimeter() float64 {
    return 2 * (r.Width + r.Height)
}

rect := Rectangle{Width: 10, Height: 5}
fmt.Println(rect.Area())       // 50
fmt.Println(rect.Perimeter())  // 30
```

### Value Receiver vs Pointer Receiver

Ini adalah salah satu perbedaan terpenting di Go:

```go
type Counter struct {
    count int
}

// VALUE receiver — receives a COPY, mutations don't stick
func (c Counter) IncrementBad() {
    c.count++  // only changes the copy
}

// POINTER receiver — receives the real struct, mutations stick
func (c *Counter) Increment() {
    c.count++  // changes the original
}

func (c Counter) Value() int {
    return c.count
}

c := Counter{}
c.IncrementBad()
fmt.Println(c.Value())  // 0 — mutation didn't stick

c.Increment()
fmt.Println(c.Value())  // 1 — mutation stuck
```

**Aturan praktis:**
- Gunakan **pointer receiver** ketika method mengubah state, atau ketika struct-nya besar
- Gunakan **value receiver** untuk operasi read-only yang kecil
- Konsisten: jika ada method pada sebuah tipe yang menggunakan pointer receiver, gunakan pointer receiver untuk semua method pada tipe tersebut

### Method pada Tipe Non-Struct

Kamu bisa mendefinisikan method pada tipe apa pun yang kamu buat — bukan hanya struct:

```go
type Celsius float64
type Fahrenheit float64

func (c Celsius) ToFahrenheit() Fahrenheit {
    return Fahrenheit(c*9/5 + 32)
}

func (f Fahrenheit) ToCelsius() Celsius {
    return Celsius((f - 32) * 5 / 9)
}

boiling := Celsius(100)
fmt.Println(boiling.ToFahrenheit())  // 212
```

### Interface Stringer

Implementasikan `String() string` untuk mengontrol cara tipe kamu dicetak:

```go
type Point struct {
    X, Y int
}

func (p Point) String() string {
    return fmt.Sprintf("(%d, %d)", p.X, p.Y)
}

p := Point{3, 4}
fmt.Println(p)        // (3, 4) — uses our String()
fmt.Printf("%v\n", p) // (3, 4)
```

# Minggu 3 — Interface & Penanganan Error

## 3.1 Interface

Sebuah interface mendefinisikan sekumpulan method. Tipe apa pun yang memiliki method-method tersebut **secara otomatis** mengimplementasikan interface itu — tidak perlu deklarasi eksplisit.

```go
// Define an interface
type Shape interface {
    Area() float64
    Perimeter() float64
}

// These types implicitly implement Shape
type Circle struct {
    Radius float64
}

func (c Circle) Area() float64 {
    return 3.14159 * c.Radius * c.Radius
}

func (c Circle) Perimeter() float64 {
    return 2 * 3.14159 * c.Radius
}

type Rectangle struct {
    Width, Height float64
}

func (r Rectangle) Area() float64 {
    return r.Width * r.Height
}

func (r Rectangle) Perimeter() float64 {
    return 2 * (r.Width + r.Height)
}
```

Sekarang fungsi apa pun yang menerima `Shape` bisa bekerja dengan keduanya:

```go
func printShapeInfo(s Shape) {
    fmt.Printf("Area: %.2f, Perimeter: %.2f\n", s.Area(), s.Perimeter())
}

printShapeInfo(Circle{Radius: 5})
printShapeInfo(Rectangle{Width: 4, Height: 6})
```

### Nilai Interface

Sebuah nilai interface punya dua komponen: tipe konkret dan nilai dari tipe tersebut.

```go
var s Shape
fmt.Println(s)        // <nil>
fmt.Println(s == nil) // true

s = Circle{Radius: 3}
fmt.Println(s)         // {3}  (uses Stringer if defined)
fmt.Println(s == nil)  // false
```

**Jebakan — nil interface vs interface yang menyimpan nilai nil:**

Ini adalah salah satu jebakan paling terkenal di Go. Sebuah interface yang menyimpan nilai konkret nil **TIDAK** sama dengan nil:

```go
type MyError struct{ Msg string }
func (e *MyError) Error() string { return e.Msg }

func mayFail(fail bool) error {
    var err *MyError  // nil pointer
    if fail {
        err = &MyError{"something broke"}
    }
    return err  // WARNING: returns a non-nil interface even when err is nil!
}

result := mayFail(false)
fmt.Println(result == nil)  // false! The interface holds (*MyError, nil) — not nil
```

**Kenapa?** Nilai interface bernilai nil hanya ketika tipe dan nilainya keduanya nil. Di sini tipe-nya adalah `*MyError` (bukan nil), meskipun nilainya nil.

**Solusi:** Kembalikan `nil` secara langsung, bukan nil pointer melalui interface:

```go
func mayFail(fail bool) error {
    if fail {
        return &MyError{"something broke"}
    }
    return nil  // explicitly return nil
}
```

### Interface Kecil Lebih Baik

Standard library Go menggunakan interface yang kecil. Yang paling terkenal:

```go
// io.Reader — just one method
type Reader interface {
    Read(p []byte) (n int, err error)
}

// io.Writer — just one method
type Writer interface {
    Write(p []byte) (n int, err error)
}

// io.ReadWriter — composed from both
type ReadWriter interface {
    Reader
    Writer
}
```

Utamakan interface dengan 1–2 method. Semakin banyak method, semakin sedikit tipe yang mengimplementasikannya.

### Aturan Emas

> **Terima interface, kembalikan struct**

```go
// BAD — accepting a concrete type limits who can call this
func SaveUser(db *PostgresDB, user User) error { ... }

// GOOD — accepting an interface lets you test with a fake DB
type Database interface {
    Save(user User) error
}
func SaveUser(db Database, user User) error { ... }

// Return structs, not interfaces (callers can always use interface)
func NewPostgresDB(url string) *PostgresDB { ... }  // return concrete type
```

## 3.2 Type Assertion & Type Switch

### Type Assertion

Mengekstrak tipe konkret dari nilai interface:

```go
var s Shape = Circle{Radius: 5}

// Assert — panics if wrong type
c := s.(Circle)
fmt.Println(c.Radius)  // 5

// Safe assert — comma-ok idiom
c, ok := s.(Circle)
if ok {
    fmt.Println("It's a circle with radius", c.Radius)
} else {
    fmt.Println("Not a circle")
}
```

### Type Switch

Menangani beberapa tipe dengan rapi:

```go
func describe(i interface{}) {
    switch v := i.(type) {
    case int:
        fmt.Printf("int: %d\n", v)
    case string:
        fmt.Printf("string: %q\n", v)
    case bool:
        fmt.Printf("bool: %v\n", v)
    case Circle:
        fmt.Printf("circle with radius %.2f\n", v.Radius)
    default:
        fmt.Printf("unknown type: %T\n", v)
    }
}

describe(42)
describe("hello")
describe(Circle{5})
```

### Tipe `any` / `interface{}`

`any` (alias untuk `interface{}`) bisa menyimpan nilai apa pun. Gunakan secukupnya — ini mengorbankan keamanan tipe.

```go
var val any = 42
val = "now a string"
val = []int{1, 2, 3}

// Before using, you must type-assert
if s, ok := val.(string); ok {
    fmt.Println("string:", s)
}
```

## 3.3 Penanganan Error

Di Go, error adalah nilai — sebuah tipe yang mengimplementasikan interface `error`:

```go
type error interface {
    Error() string
}
```

### Pola Dasar

```go
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("division by zero")
    }
    return a / b, nil
}

result, err := divide(10, 0)
if err != nil {
    fmt.Println("Error:", err)
    return
}
fmt.Println(result)
```

Pola ini muncul di mana-mana. Biasakan diri kamu dengannya sesegera mungkin.

### Membuat Error

```go
import (
    "errors"
    "fmt"
)

// Simple error
err1 := errors.New("something went wrong")

// Formatted error
name := "config.json"
err2 := fmt.Errorf("file not found: %s", name)

// Wrapping an error — preserves the original for inspection
original := errors.New("connection refused")
wrapped := fmt.Errorf("failed to connect to database: %w", original)
```

### Wrapping & Unwrapping Error

Gunakan `%w` untuk membungkus error, dan `errors.Is` / `errors.As` untuk memeriksa:

```go
var ErrNotFound = errors.New("not found")

func findUser(id int) (*User, error) {
    if id <= 0 {
        return nil, fmt.Errorf("findUser(%d): %w", id, ErrNotFound)
    }
    // ...
}

err := findUser(-1)

// errors.Is checks if any error in the chain matches
if errors.Is(err, ErrNotFound) {
    fmt.Println("User was not found")
}

// errors.As extracts a specific type from the chain
var notFound *NotFoundError
if errors.As(err, &notFound) {
    fmt.Println("Resource:", notFound.Resource)
}
```

### Tipe Error Kustom

Definisikan struct yang mengimplementasikan interface `error`:

```go
type ValidationError struct {
    Field   string
    Message string
}

func (e *ValidationError) Error() string {
    return fmt.Sprintf("validation error on field %q: %s", e.Field, e.Message)
}

func validateAge(age int) error {
    if age < 0 {
        return &ValidationError{Field: "age", Message: "must be non-negative"}
    }
    if age > 150 {
        return &ValidationError{Field: "age", Message: "unrealistically large"}
    }
    return nil
}

err := validateAge(-5)
var ve *ValidationError
if errors.As(err, &ve) {
    fmt.Println("Field:", ve.Field)    // age
    fmt.Println("Message:", ve.Message) // must be non-negative
}
```

### panic & recover

`panic` menghentikan eksekusi normal dan membuka (unwind) stack. Gunakan jarang — hanya untuk situasi yang benar-benar tidak bisa dipulihkan (bug programmer, bukan error dari pengguna).

```go
func mustPositive(n int) int {
    if n <= 0 {
        panic(fmt.Sprintf("expected positive, got %d", n))
    }
    return n
}

// Recover from a panic (e.g., in a web server handler)
func safeCall(fn func()) (err error) {
    defer func() {
        if r := recover(); r != nil {
            err = fmt.Errorf("recovered from panic: %v", r)
        }
    }()
    fn()
    return nil
}
```

**Aturan:** Kembalikan error untuk kasus kegagalan yang diharapkan. Gunakan panic untuk bug yang seharusnya tidak pernah terjadi.

# Minggu 4 — Package, Module & Testing

## 4.1 Package

Setiap file Go milik sebuah package. Package adalah unit dasar pengorganisasian kode.

```go
// math/calc.go
package math   // package declaration

// Exported — starts with uppercase
func Add(a, b int) int {
    return a + b
}

// Unexported — starts with lowercase, only accessible within this package
func clamp(n, min, max int) int {
    if n < min { return min }
    if n > max { return max }
    return n
}
```

### Aturan Penamaan Package

- Pendek, huruf kecil, tanpa underscore, tanpa mixedCase
- Nama package harus sesuai dengan nama direktori
- Gunakan bentuk tunggal: `user` bukan `users`, `model` bukan `models`
- Hindari nama generik: `util`, `common`, `helper` adalah tanda bahaya

### Mengimpor Package

```go
import "fmt"
import "os"

// Block import
import (
    "fmt"
    "os"
    "strings"

    // Third-party packages use their module path
    "github.com/gin-gonic/gin"
)

// Import alias
import (
    "fmt"
    myfmt "github.com/mylib/fmt"  // alias to avoid conflict
    _ "database/sql/driver"       // blank import — runs init() only
)
```

### Fungsi `init()`

Setiap package bisa punya satu atau lebih fungsi `init()`. Fungsi ini berjalan otomatis sebelum `main()`, sesuai urutan package diimpor.

```go
package database

import "fmt"

func init() {
    fmt.Println("database package initialized")
    // register drivers, set defaults, validate config, etc.
}
```

**Kenapa blank import ada:** Beberapa package hanya memerlukan `init()` mereka untuk dijalankan. Misalnya, driver database mendaftarkan dirinya sendiri di `init()`:

```go
import (
    "database/sql"
    _ "github.com/lib/pq"  // registers the PostgreSQL driver via init()
)

// Now you can use:
db, err := sql.Open("postgres", connString)
```

Tanpa blank import, Go tidak akan mengompilasi (import yang tidak terpakai adalah error), dan driver tidak akan pernah mendaftarkan dirinya.

**Aturan:** Jangan berlebihan menggunakan `init()`. Fungsi ini berjalan secara implisit, membuat kode lebih sulit diikuti. Utamakan inisialisasi eksplisit di `main()` jika memungkinkan.

Go punya package testing bawaan. Tidak perlu framework eksternal.

### Test Dasar

File harus diakhiri dengan `_test.go`. Fungsi test harus dimulai dengan `Test` dan menerima `*testing.T`:

```bash

```

### Table-Driven Test

Pola standar Go — tulis satu test dengan banyak kasus:

```go
func TestAdd(t *testing.T) {
    tests := []struct {
        name     string
        a, b     int
        expected int
    }{
        {"positive numbers", 2, 3, 5},
        {"negative numbers", -1, -2, -3},
        {"zeros", 0, 0, 0},
        {"mixed", -5, 10, 5},
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            got := Add(tt.a, tt.b)
            if got != tt.expected {
                t.Errorf("Add(%d, %d) = %d; want %d", tt.a, tt.b, got, tt.expected)
            }
        })
    }
}
```

`t.Run` membuat subtest — kamu bisa menjalankan subtest tertentu:

```bash
go test -run TestAdd/negative_numbers
```

### Testing Error

```go
func TestDivide(t *testing.T) {
    tests := []struct {
        name        string
        a, b        float64
        expected    float64
        expectError bool
    }{
        {"normal division", 10, 2, 5.0, false},
        {"divide by zero", 10, 0, 0, true},
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            result, err := Divide(tt.a, tt.b)
            if tt.expectError {
                if err == nil {
                    t.Error("expected an error but got nil")
                }
                return
            }
            if err != nil {
                t.Errorf("unexpected error: %v", err)
                return
            }
            if result != tt.expected {
                t.Errorf("got %f; want %f", result, tt.expected)
            }
        })
    }
}
```

```go
        t.Fatal("expected an error but got nil")
    }
}
```

# Minggu 5 — Goroutine & Channel

## 5.1 Goroutine

Goroutine adalah thread ringan yang dikelola oleh runtime Go. Membuatnya sangat murah (~2KB ruang stack, bertambah sesuai kebutuhan).

```go
func say(msg string) {
    for i := 0; i < 3; i++ {
        fmt.Println(msg)
        time.Sleep(100 * time.Millisecond)
    }
}

func main() {
    go say("world")  // starts a new goroutine
    say("hello")     // runs in the main goroutine
}
```

**Masalah:** Jika `main` return, SEMUA goroutine langsung dimatikan.

```go
func main() {
    go fmt.Println("I might not print!")  // main exits before this runs
}
```

### sync.WaitGroup — Menunggu Goroutine

```go
import "sync"

func main() {
    var wg sync.WaitGroup

    for i := 0; i < 5; i++ {
        wg.Add(1)                          // increment counter before launching
        go func(id int) {
            defer wg.Done()                // decrement when goroutine finishes
            fmt.Printf("Worker %d done\n", id)
        }(i)                               // pass i as argument — avoid closure trap
    }

    wg.Wait()  // block until counter reaches 0
    fmt.Println("All workers done")
}
```

**Jebakan closure:** Hati-hati menangkap variabel loop di goroutine:

```go
// BUG — all goroutines share the same 'i' variable
for i := 0; i < 3; i++ {
    go func() {
        fmt.Println(i)  // might print 3 3 3
    }()
}

// FIX — pass 'i' as an argument to create a copy
for i := 0; i < 3; i++ {
    go func(i int) {
        fmt.Println(i)  // prints 0, 1, 2 (in some order)
    }(i)
}
```

## 5.2 Channel

Channel adalah saluran bertipe untuk komunikasi antar goroutine. Ini adalah cara yang diutamakan untuk berbagi data.

```go
// Create a channel
ch := make(chan int)       // unbuffered
bch := make(chan int, 5)   // buffered with capacity 5

// Send a value (blocks if receiver isn't ready — unbuffered)
ch <- 42

// Receive a value (blocks until a value is sent)
val := <-ch

// Close a channel — signals no more values will be sent
close(ch)

// Check if channel is closed (comma-ok)
val, ok := <-ch
if !ok {
    fmt.Println("channel closed")
}
```

### Unbuffered vs Buffered

```go
// Unbuffered — sender blocks until receiver is ready (synchronous)
ch := make(chan int)
go func() {
    ch <- 1  // blocks here until main receives
}()
val := <-ch  // unblocks the goroutine
fmt.Println(val)

// Buffered — sender blocks only when buffer is full (asynchronous up to capacity)
bch := make(chan int, 3)
bch <- 1  // doesn't block
bch <- 2  // doesn't block
bch <- 3  // doesn't block
// bch <- 4  // would block — buffer full

fmt.Println(<-bch)  // 1
fmt.Println(<-bch)  // 2
fmt.Println(<-bch)  // 3
```

### Iterasi Channel dengan Range

```go
func generate(nums ...int) <-chan int {
    out := make(chan int)
    go func() {
        for _, n := range nums {
            out <- n
        }
        close(out)  // must close to end the range loop
    }()
    return out
}

for n := range generate(1, 2, 3, 4, 5) {
    fmt.Println(n)
}
```

### Channel Berarah (Directional)

Membatasi apa yang bisa dilakukan fungsi dengan sebuah channel:

```go
func producer(out chan<- int) {  // send-only
    out <- 42
}

func consumer(in <-chan int) {   // receive-only
    val := <-in
    fmt.Println(val)
}

ch := make(chan int)
go producer(ch)
consumer(ch)
```

## 5.3 Statement select

`select` memungkinkan goroutine menunggu beberapa operasi channel secara bersamaan:

```go
ch1 := make(chan string)
ch2 := make(chan string)

go func() {
    time.Sleep(1 * time.Second)
    ch1 <- "one"
}()
go func() {
    time.Sleep(2 * time.Second)
    ch2 <- "two"
}()

// Whichever channel is ready first — that case runs
select {
case msg := <-ch1:
    fmt.Println("Received from ch1:", msg)
case msg := <-ch2:
    fmt.Println("Received from ch2:", msg)
}
```

### Done Channel — Pembatalan

```go
func worker(done <-chan struct{}) {
    for {
        select {
        case <-done:
            fmt.Println("worker: stopping")
            return
        default:
            fmt.Println("worker: working...")
            time.Sleep(500 * time.Millisecond)
        }
    }
}

func main() {
    done := make(chan struct{})
    go worker(done)

    time.Sleep(2 * time.Second)
    close(done)  // signal all workers to stop
    time.Sleep(500 * time.Millisecond)
}
```

## 5.4 Mutex & Shared State

Ketika goroutine berbagi memori alih-alih berkomunikasi lewat channel, lindungi dengan mutex:

```go
import "sync"

type SafeCounter struct {
    mu    sync.Mutex
    count int
}

func (c *SafeCounter) Increment() {
    c.mu.Lock()
    defer c.mu.Unlock()
    c.count++
}

func (c *SafeCounter) Value() int {
    c.mu.Lock()
    defer c.mu.Unlock()
    return c.count
}

// Usage
counter := SafeCounter{}
var wg sync.WaitGroup

for i := 0; i < 100; i++ {
    wg.Add(1)
    go func() {
        defer wg.Done()
        counter.Increment()
    }()
}

wg.Wait()
fmt.Println(counter.Value())  // 100 — always
```

### RWMutex — Banyak Reader, Satu Writer

```go
type Cache struct {
    mu   sync.RWMutex
    data map[string]string
}

// Multiple goroutines can read at the same time
func (c *Cache) Get(key string) (string, bool) {
    c.mu.RLock()
    defer c.mu.RUnlock()
    val, ok := c.data[key]
    return val, ok
}

// Only one goroutine can write at a time
func (c *Cache) Set(key, val string) {
    c.mu.Lock()
    defer c.mu.Unlock()
    c.data[key] = val
}
```

### Race Detector

Selalu jalankan dengan `-race` selama pengembangan:

```bash
go run -race main.go
go test -race ./...
```

Tool ini menginstrumentasi binary kamu untuk mendeteksi data race saat runtime. Jika ia melaporkan sesuatu, itu bug yang nyata.

# Minggu 6 — Konsolidasi & Proyek Akhir

## 6.1 Generics (Go 1.18+)

Generics memungkinkan kamu menulis fungsi dan tipe yang bekerja dengan berbagai tipe.

```go
// Before generics — you'd need separate functions or use interface{}
func SumInts(nums []int) int {
    total := 0
    for _, n := range nums { total += n }
    return total
}

func SumFloat64s(nums []float64) float64 {
    total := 0.0
    for _, n := range nums { total += n }
    return total
}

// With generics — one function for both
func Sum[T int | float64](nums []T) T {
    var total T
    for _, n := range nums { total += n }
    return total
}

Sum([]int{1, 2, 3})           // 6
Sum([]float64{1.1, 2.2, 3.3}) // 6.6
```

### Constraint

```go
import "cmp"  // standard library since Go 1.21

// Built-in constraints
func Max[T cmp.Ordered](a, b T) T {
    if a > b { return a }
    return b
}

// Note: Before Go 1.21, you had to use "golang.org/x/exp/constraints"
// which is now deprecated. Always use "cmp.Ordered" from the standard library.

// Custom constraint
type Number interface {
    int | int8 | int16 | int32 | int64 |
    float32 | float64
}

func Abs[T Number](n T) T {
    if n < 0 { return -n }
    return n
}
```

### Struktur Data Generik

```go
type Stack[T any] struct {
    items []T
}

func (s *Stack[T]) Push(item T) {
    s.items = append(s.items, item)
}

func (s *Stack[T]) Pop() (T, bool) {
    var zero T
    if len(s.items) == 0 {
        return zero, false
    }
    top := s.items[len(s.items)-1]
    s.items = s.items[:len(s.items)-1]
    return top, true
}

// Type-safe stacks for any type
intStack := Stack[int]{}
strStack := Stack[string]{}
```

**Kapan menggunakan generics vs kapan menggunakan interface:**
- **Gunakan generics** untuk struktur data (Stack, Set, Map), fungsi utilitas (Min, Max, Filter, Map), dan ketika kamu butuh koleksi yang type-safe.
- **Gunakan interface** untuk abstraksi perilaku (Reader, Writer, Handler), dependency injection, dan ketika tipe yang berbeda memerlukan implementasi yang berbeda.
- **Jangan gunakan generics** hanya karena bisa. Jika interface sudah cukup, gunakan interface. Generics menambah kompleksitas — pastikan ada alasan kuat.

## 6.2 Pola-Pola Umum Go

### Functional Options

Menyelesaikan masalah struct dengan banyak field konfigurasi opsional:

```go
type Server struct {
    host    string
    port    int
    timeout time.Duration
}

type Option func(*Server)

func WithHost(host string) Option {
    return func(s *Server) {
        s.host = host
    }
}

func WithPort(port int) Option {
    return func(s *Server) {
        s.port = port
    }
}

func WithTimeout(d time.Duration) Option {
    return func(s *Server) {
        s.timeout = d
    }
}

func NewServer(opts ...Option) *Server {
    s := &Server{
        host:    "localhost",   // defaults
        port:    8080,
        timeout: 30 * time.Second,
    }
    for _, opt := range opts {
        opt(s)
    }
    return s
}

// Clean call site
s := NewServer(
    WithPort(9090),
    WithTimeout(10 * time.Second),
)
```

### sync.Once — Jalankan Tepat Sekali

```go
var (
    instance *DB
    once     sync.Once
)

func GetDB() *DB {
    once.Do(func() {
        instance = connectToDB()
    })
    return instance
}
```

Aman untuk penggunaan konkuren — `connectToDB` dipanggil tepat sekali, bahkan jika `GetDB` dipanggil dari beberapa goroutine secara bersamaan.

### Terima Interface, Kembalikan Struct

Prinsip ini membuat kode kamu fleksibel dan mudah ditest:

```go
// GOOD — accepts interface (flexible, testable)
func ProcessOrders(repo OrderReader) error {
    orders, err := repo.GetPending()
    // ...
}

// GOOD — returns concrete type (clear, easy to use)
func NewOrderService(db *sql.DB) *OrderService {
    return &OrderService{db: db}
}
```

- **Terima interface:** Pemanggil bisa memberikan implementasi apa pun (DB asli, fake untuk testing).
- **Kembalikan struct:** Pemanggil tahu persis apa yang mereka dapat — tidak perlu menebak apa yang ada di balik interface.
